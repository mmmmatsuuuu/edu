    第6.5時：反復構造の発展（while文）／予備・可変

    この回の位置づけ（重要）：本時は時間に余裕がある場合に実施する予備回。第6回までの進度に余裕がなければ実施せず、本時の要点（①for↔whileの等価性／②使い分け／③無限ループの注意）を第6回のまとめで5〜10分に圧縮して紹介する。どちらの場合も、while を「自分で書ける」よりまず「読めて・落とし穴を知っている」状態を目標とする。

    本時の目標

    for と while の違い（回数が決まっている／条件で続ける）を理解し、回数が事前に決まらない繰り返しを while で表現できるようになる。while 特有の無限ループの落とし穴を理解し、ループ変数の更新で回避できる。

    キーワード

    用語 説明
    while文 継続条件が真の間、処理を繰り返す構文。回数が事前に決まらない繰り返しに向く
    継続条件 ループを続けるかどうかを判定する真偽の式（合計 < 目標 など）
    ループ変数の更新 ループ内で条件を偽に近づける処理（カウンタの増分・入力の受け直しなど）
    無限ループ 継続条件が永遠に偽にならず、ループが止まらなくなること
    番兵（sentinel） 入力の終わりを表す特別な値（-1 など）。これが来たら反復を止める
    for↔whileの等価性 回数の決まった反復は for でも while でも同じ結果を書ける

    準備物・教材

    • PRIMワークシート（Predict/Run/Investigate/Modify 欄・実行環境リンク付き）※未実施時は第6回の圧縮紹介用スライドに転用
    • 種コード：第6回の「1〜10の合計」を while で書き換えた等価版／しきい値・番兵で止まる while の例
    • オンライン実行環境（Google Colab 等。無限ループを止める操作も確認できる環境）

    授業展開

    時間 段階 学習活動 指導上の留意点
    5分 導入 第6回の for による合計を振り返り、「回数が決まらない繰り返し（例：合計が目標を超えるまで）はどう書くか?」と問いかける for は「回数・範囲が決まっている」反復だったことを確認し、対比で入る
    10分 Predict 第6回の合計を while で書いた等価版を読み、出力とループ変数（カウンタ・合計）の各回の変化を予想する for版と見比べ、「初期化・継続条件・更新」がどこに対応するか注目させる
    8分 Run 実行して予想と照合する。続けてループ変数の更新を消したコードを実演し、無限ループになる様子と止め方を見せる 無限ループは必ず教師が実演で見せる。「止まらない＝条件が偽にならない」原因をその場で言語化する
    12分 Investigate while の「初期化／継続条件／更新」の3点セットと for との対応を調べる。使い分け（回数が決まる→for、条件で続ける→while）を整理する 「更新を書く責任は自分にある」点を for との最大の違いとして強調する
    12分 Modify 等価版を「合計が目標値を超えるまで加算し、何回で超えたかを求める」／「番兵 -1 が来るまで入力を合計する」に書き換える 終了条件と更新を意識させる。1か所ずつ変えて実行し、止まることを必ず確認させる
    3分 まとめ while を使うときは「条件を必ず偽に近づける更新を書く」を鉄則として板書する。今後 while を自習で使う際の注意として渡す 「読めて・使い分けが分かり・無限ループを避けられる」状態を確認して終える

    板書・スライド構成

    1. 今日の問い：回数が決まらない繰り返しはどう書く?
    2. for↔while の等価性（合計を両方で・3点セットの対応）
    3. while の3点セット：初期化／継続条件／更新
    4. 落とし穴：更新を忘れると無限ループ（実演）
    5. 使い分け：回数が決まる→for／条件で続ける→while
    6. まとめ：更新を必ず書く（自習時の鉄則）

    本時の評価

    観点 評価方法 内容
    知識・技能 ワークシート（Investigate） while の3点セットと for との対応、無限ループの原因を説明できる
    思考・判断・表現 ワークシート（Modify） 条件で止まる反復（しきい値・番兵）を while で書き換えられる
