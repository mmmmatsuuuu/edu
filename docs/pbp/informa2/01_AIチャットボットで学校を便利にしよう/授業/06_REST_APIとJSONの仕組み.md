# 第6時：REST APIとJSONの仕組み

## 本時の目標

1. REST APIの概念（エンドポイント・HTTPメソッド・リクエスト・レスポンス）を説明できる
2. JSONのデータ構造（オブジェクト・配列・ネスト）を読み書きできる
3. ブラウザや簡単なツールを使ってAPIリクエストを送り、レスポンスを確認できる

## 取り扱う学習指導要領の内容

- エ-(ア) 情報システムの設計と開発
- エ-(ウ) プログラミング

## 準備物・教材

- 教師：スライド、Webブラウザ（DevTools）、Postmanまたはhttpie、公開API（OpenWeatherMap等）のデモ
- 生徒：PC（ブラウザ・Postman利用）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 10分 | 導入 | 「レストランに例えるAPI」の比喩で概念を導入。客（クライアント）・メニュー（API仕様）・ウエイター（API）・料理（データ）のアナロジー | 比喩を板書に図示し、抽象的な概念を具体化する |

## 板書・スライド構成

- スライド1：レストランに例えるAPI（図解）
- スライド2：HTTPメソッドの4種類と役割
- スライド3：JSONの構造（オブジェクト・配列・ネスト）
- スライド4：Postman操作手順（スクリーンショット）
- スライド5：本日使ったAPIのエンドポイントとレスポンス例

## 本時の評価

- 観点：知識・技能
- 評価方法：JSONクイズの正答状況
- 評価基準：ネスト構造を含むJSONから指定されたフィールドの値を正確に取り出せる
