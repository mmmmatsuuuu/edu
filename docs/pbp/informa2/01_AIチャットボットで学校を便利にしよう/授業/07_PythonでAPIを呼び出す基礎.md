# 第7時：PythonでAPIを呼び出す基礎

## 本時の目標

1. Pythonのrequestsライブラリを使ってHTTP GETリクエストを送信し、レスポンスを取得できる
2. 環境変数を使ってAPIキーを安全に管理する方法を理解・実践できる
3. JSONレスポンスをPythonの辞書として扱い、必要なデータを取り出せる

## 取り扱う学習指導要領の内容

- エ-(ウ) プログラミング
- ア-(イ) 情報に関する法規・倫理・セキュリティ

## 準備物・教材

- 教師：Google Colabノートブック（雛形）、APIキー（教師用デモ用）
- 生徒：PCとGoogleアカウント（Google Colab使用）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 10分 | 導入 | 前時のPostmanデモをPythonで再現する動機付け。「たった5行でAPIを叩ける」というライブコーディングデモを教師が行う | 短いコードで大きなことができるという驚きを体験させる |
| 30分 | 展開 | ①requestsライブラリのインポートとGETリクエストの基本構文を解説（5分）②ハンズオン：天気APIまたは教師作成のデモAPIに対してGETリクエストを送る（10分）③レスポンスの.json()メソッドで辞書化→必要データを取り出す（5分）④環境変数の概念とColabでのシークレット設定（5分）⑤OpenAI API（または類似API）に簡単なテキストを送ってレスポンスを受け取る（5分） | APIキーをコードにハードコードしない習慣をこの時点で徹底する。セキュリティ意識の醸成が重要 |
| 10分 | まとめ | コードをColabに保存して提出。振り返りシートに「できたこと」「詰まったこと」を記入 | エラーが出た生徒には共通エラー一覧を参照させる |

## 板書・スライド構成

- スライド1：前時の復習（REST API・JSON）
- スライド2：requestsライブラリの基本コード（annotated）
- スライド3：APIキーの安全な管理（環境変数 vs ハードコード）
- スライド4：よくあるエラーと対処法（404・401・timeout）
- スライド5：本日のミニ課題（APIから〇〇を取り出して表示せよ）

## 本時の評価

- 観点：知識・技能
- 評価方法：ColabノートブックのコードとAPIレスポンスの確認
- 評価基準：requestsを使ってAPIを呼び出し、JSONから指定データを取り出すコードが動作している
