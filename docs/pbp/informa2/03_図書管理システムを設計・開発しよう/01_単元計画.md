# 単元名：図書管理システムを設計・開発しよう

## 単元の目標

1. システム開発の工程（要件定義→設計→実装→テスト）を理解し、一連の流れを経験できる
2. データベースの正規化とER図を用いた設計ができ、SQLで基本的なCRUD操作を実装できる
3. オブジェクト指向プログラミングの概念（クラス・継承・カプセル化）を理解し、Pythonで実装できる
4. Gitの基本操作を理解し、バージョン管理の重要性を説明できる

## 授業時数

全14時間

## 取り扱う学習指導要領の内容

- エ-(ア) 情報システムの設計と開発
- エ-(イ) データベースの設計と活用
- エ-(ウ) プログラミング
- オ 情報と情報技術を活用した問題発見・解決の探究

## 単元の評価基準

| 観点 | 評価基準 | 評価方法 |
|---|---|---|
| 知識・技能 | SQL・OOP・Gitの基本操作を正しく実装できる | コードレビュー・演習課題 |
| 思考・判断・表現 | 要件を分析してER図・クラス設計を作成し、設計の意図を説明できる | 設計書・発表 |
| 主体的に学習に取り組む態度 | エラーに粘り強く対処しながらグループ開発に積極的に参加している | 観察・進捗記録 |
