# 第12時：バージョン管理（Gitの概念と基本操作）

## 本時の目標

1. バージョン管理の必要性を理解し、Gitの基本概念（リポジトリ・コミット・ブランチ）を説明できる
2. git init・add・commit・statusの基本コマンドを実行できる
3. GitHubにリポジトリを作成してpushし、コードを共有できる

## 取り扱う学習指導要領の内容

- エ-(ア) 情報システムの設計と開発
- エ-(ウ) プログラミング

## 準備物・教材

- 教師：Git基礎スライド、GitHub操作デモ
- 生徒：PC、GitHubアカウント（事前作成推奨）、ターミナルまたはGitHub Desktop

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 10分 | 導入 | 「バージョン管理なしの開発」の悲劇：「final_v2_最終版_本当の最終.py」というファイル名で笑いを取りながら、管理の難しさを実感させる | 身近な問題（課題ファイルの命名）と対応させることで共感を得る |
| 30分 | 展開 | ①Gitの概念図（ワークツリー→ステージング→ローカルリポジトリ→リモートリポジトリ）（5分）②ターミナルで実際にコマンドを実行：git init → git add → git commit（10分）③ブランチの概念（main vs 開発ブランチ）（5分）④GitHubでリポジトリ作成 → git remote add → git push（10分） | Gitはエラーメッセージが難解なため、コマンドの順番を手順書として提供する。GitHub Desktopでも可 |
| 10分 | まとめ | 図書管理システムのコードをGitHubにpushして保存。振り返りシートに「Gitを使えばどんな問題が解決するか」を記入 | 現代のソフトウェア開発でGitHubが標準ツールであることを伝え、ポートフォリオとしての価値にも触れる |

## 板書・スライド構成

- スライド1：バージョン管理の失敗例（笑えるファイル名）
- スライド2：Gitのデータフロー図（4つの領域）
- スライド3：基本コマンド一覧（init / add / commit / push）
- スライド4：GitHubでのリポジトリ作成手順
- スライド5：ブランチとマージの概念図

## 本時の評価

- 観点：知識・技能
- 評価方法：GitHubへのpushの確認
- 評価基準：コードをGitHubにpushし、コミットメッセージに作業内容が記述されている
