# 第6時：SQLの応用（UPDATE・DELETE・JOIN）

## 本時の目標

1. UPDATE・DELETE文を適切なWHERE条件と組み合わせて使用できる
2. INNER JOIN・LEFT JOINを使って複数テーブルを結合してデータを取得できる
3. 副問い合わせ（サブクエリ）の基本的な使い方を理解できる

## 取り扱う学習指導要領の内容

- エ-(イ) データベースの設計と活用
- エ-(ウ) プログラミング

## 準備物・教材

- 教師：SQL応用演習Colab、SQLクイズシート
- 生徒：PC、前時作成のSQLファイル・DBファイル

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 10分 | 導入 | 「WHERE条件なしにDELETEするとどうなる？」：全データが消えるデモを見せる。「本番DBではトランザクションが重要」という話で気をつける点を伝える | インパクトのある失敗例を見せることで、慎重な操作の習慣を身につけさせる |
| 30分 | 展開 | ①UPDATE・DELETE文の構文とWHERE条件の重要性（5分）②INNER JOINの概念（2つのテーブルの共通部分）と構文（10分）③LEFT JOINの概念（左テーブル全件+右テーブルの一致分）（5分）④貸出記録と本・利用者を結合する演習（5分）⑤副問い合わせの基本（IN・EXISTS）（5分） | JOINは図解が効果的。集合の「重なり」として可視化する |
| 10分 | まとめ | SQLクイズ（「現在貸出中の本の一覧を利用者名・本のタイトル付きで取得せよ」などJOINを使う問題3問）。振り返りシートに記入 | 次時からPythonでこのSQLを実行する実装フェーズに入ることを予告する |

## 板書・スライド構成

- スライド1：WHERE条件なしDELETEのデモ（インパクト）
- スライド2：UPDATE・DELETE構文と安全な使い方
- スライド3：INNER JOIN vs LEFT JOINの図解（集合図）
- スライド4：JOINのSQL構文例（図書管理の文脈で）
- スライド5：SQLクイズ問題

## 本時の評価

- 観点：知識・技能
- 評価方法：SQLクイズ（JOIN含む複数テーブル検索）
- 評価基準：JOINを使って複数テーブルから必要なデータを正確に取得できる
