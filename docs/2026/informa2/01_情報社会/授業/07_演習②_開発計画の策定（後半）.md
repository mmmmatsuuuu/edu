# 第7時：演習②「開発計画の策定（後半）」

## 本時の目標

- フィジビリティを検討し、実現が難しい点と解決の方向性を整理できる
- スケジュールと役割分担を含む開発計画書を完成させられる
- 計画書をグループで発表し、他グループの視点を踏まえて改善点を整理できる

## 準備物・教材

【教師】
- 演習説明スライド（前時の振り返り・後半の作業範囲の説明）
- 開発計画書テンプレート（前時から継続）

【生徒】
- 学習用端末
- 前時（演習①）で作成中の開発計画書

## 開発計画書テンプレートの構成（後半で記入する範囲）

```
■ フィジビリティの確認
（実現が難しいと思う点・解決策の案）

■ スケジュール（案）
| 時期 | 単元 | 主な作業 |
|---|---|---|
| 6〜7月 | コンテンツ単元 | 景品コンテンツの制作 |
| 9〜10月 | 情報システム単元 | システムの設計・構築 |
| 11月 | 文化祭 | 本番運用 |
| 12〜2月 | データサイエンス単元 | 運用データの分析 |

■ 役割分担案
（コンテンツ担当・システム担当・データ分析担当等）
```

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 前時の開発計画書（前半）を確認し、今日完成させる範囲（フィジビリティ・スケジュール・役割分担）を確認する | 前時で「必ずやる機能」として挙げたものが今日のフィジビリティ検討の出発点になることを伝える |
| 30分 | 作業 | グループで開発計画書（後半）を作成する。①フィジビリティの確認（前時の要件に対して、技術面・時間面・スキル面での課題を挙げ、解決の方向性を書く）②スケジュールの確認（年間を通じた作業の流れを把握し、各単元で何をするかを整理する）③役割分担案の検討（コンテンツ・システム・データ分析の3領域を意識して分担を考える） | フィジビリティは「できない理由」を探すのではなく「どうすれば実現できるか」の視点で考えさせる。スケジュールは年間計画を見せながら、どの単元でどの作業をするかを対応づけさせる。役割分担は「やりたいこと」だけでなく「プロジェクト全体のバランス」も考えさせる |
| 15分 | 発表・振り返り | 各グループが開発計画書の要点（実現したい機能・フィジビリティ上の課題）を1〜2分で共有する。発表後、全体で気づいたことを共有し、振り返りフォームに記入する | 他グループの発表から「自分たちが見落としていた視点」を拾わせる。単元の学びがプロジェクトの法的基盤・問題解決の計画立案につながったことを確認する。開発計画書を提出させてまとめとする |

## 板書・スライド構成

1. 前時の振り返り・今日の作業範囲
2. フィジビリティ検討の視点（技術面・時間面・スキル面）
3. 年間スケジュールの全体像（各単元と対応する作業）
4. 役割分担の3領域（コンテンツ・システム・データ分析）
5. 発表のポイント（実現したい機能・フィジビリティ上の課題を中心に）

## 本時の評価

- **思考・判断・表現**：開発計画書の完成版から、要件定義・フィジビリティ・スケジュール・役割分担が一貫した計画として整理されているか評価（本時終了後に提出）
- **主体的に学習に取り組む態度**：グループ内での積極的な議論・発表への参加

## 対応する学習指導要領の内容

- 情報Ⅱ（1）ア：情報技術の発展と社会の変化（情報技術を活用した問題解決）
