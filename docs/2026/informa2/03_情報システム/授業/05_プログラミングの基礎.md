# 第5時：プログラミングの基礎（復習＋GAS/JS）

## 本時の目標

- 1年次に学んだプログラミングの基本3構造（順次・分岐・反復）を思い出し、変数・条件分岐・繰り返し・関数の役割を説明できる。
- GASがJavaScriptであることを理解し、PythonとJavaScriptの構文の違い（波括弧・セミコロン・コロンの有無など）に気づく。
- 短いGAS/JSコードを PRIMM（Predict→Run→Investigate→Modify、Makeなし）で読み解き、「何をしているか」を説明できる。

## キーワード

| 用語 | 説明 |
|---|---|
| 変数 | データに名前を付けて入れておく箱。`let`/`const` で用意する |
| 基本3構造（順次・分岐・反復） | プログラムの基本的な流れ。上から順に・条件で分かれ・繰り返す |
| 条件分岐（if） | 条件が成り立つかどうかで処理を変える仕組み |
| 繰り返し（for） | 同じ処理を決まった回数や条件の間くり返す仕組み |
| 関数（function） | 一連の処理にまとめて名前を付けたもの。呼び出して再利用できる |
| 配列・オブジェクト | 複数のデータをまとめて扱う入れ物（配列＝並び、オブジェクト＝名前付きの組） |
| GAS（Google Apps Script） | Googleのサービス上でJavaScriptを動かす仕組み。スプレッドシートと連携したWebアプリを作れる |
| PRIMM | Predict→Run→Investigate→Modify→Make の学習サイクル。本時はModifyまで（Makeなし）でコードを読む力をつける |

## 準備物・教材

- 教師：授業動画（基本3構造の復習・GAS＝JavaScript・JSの基本構文・PythonとJSの対比）、スライド
- 教師：PRIMMワークシート（Predict／Run／Investigate／Modify の記入欄＋実行環境リンク付き。情報Ⅰの様式に準拠）
- 教師：題材コード（スタンプ処理風の短いGAS/JS。例：スタンプ数を数える `for`、ガチャ可否を判定する `if`、まとまった処理の `function`）
- 生徒：1人1台端末、GAS／JS実行環境（Apps Scriptエディタ等）、Google Classroom
- 教科書：情報Ⅱ（東京書籍）第4章

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 前時の復習（動的サイト＝Webアプリ）。「その中身のプログラムはどう書かれているのか」を問いかけ、1年次のプログラミングを思い出す | 「書けるより読める」を本時のゴールとして明示し、不安を下げる |
| 18分 | 展開①（動画視聴・インプット） | 授業動画を視聴：基本3構造（順次・分岐・反復）の復習、変数・`if`・`for`・`function`・配列/オブジェクト、GAS＝JavaScriptであること、**PythonとJSの対比**（波括弧でブロック・行末セミコロン・`:`が無い・`let`/`const`での変数宣言など） | 動画はアプリには触れず一般知識として扱う。Pythonとの違いを表で対比し、つまずきやすい記号（`{}` `;`）を予防的に強調する |
| 20分 | 展開②（確認・深化／PRIMM・Makeなし） | スタンプ処理風の短いGAS/JSコードを題材に PRIMM を回す。**Predict**：出力／挙動を予想 → **Run**：実行して確認しズレをメモ → **Investigate**：`let`/`if`/`for`/`function` の意味を調べて確認（Python対比に接続）→ **Modify**：一部（条件の数値や繰り返し回数など）だけ書き換えて変化を見る。**Make（ゼロから記述）は行わない** | 予想を必ず表明させてから実行させる。Investigateで「なぜそう動くか」を言語化。Modifyは小さな変更に限定し、コードを読む力に集中させる。詰まった生徒を個別フォロー |
| 7分 | まとめ | キーワード整理。「AIが書いたコードも、構造（変数・分岐・反復・関数）に注目すれば読める」ことを確認。次時（APIと生成AIの活用）へ接続 | 後の演習で「AI出力を読んで検証する」前提になることを意識づける |

## 板書・スライド構成

1. 本時のテーマ：プログラミングの基礎（復習＋GAS/JS）／ゴール＝「読める」
2. 基本3構造の復習：順次・分岐・反復
3. JSの基本：変数（`let`/`const`）・`if`・`for`・`function`・配列/オブジェクト
4. PythonとJSの対比表（`{}`・`;`・`:`の有無・宣言のしかた）
5. GAS＝Google上で動くJavaScript（スプレッドシート連携）
6. ＜PRIMM＞題材コードと Predict→Run→Investigate→Modify の進め方（Makeなし）

## 本時の評価

- **観点：知識・技能**（評価方法：PRIMMワークシート・授業内の発言）
  - 基本3構造とJSの基本構文（変数・`if`・`for`・`function`）を説明できているか。
  - 短いGAS/JSコードを読んで「何をしているか」を説明できているか（Predict・Investigateの記述）。
- 形成的評価とし、以降の演習（AI出力コードの検証）と単元テストにつなげる。

## 備考・メモ

- 1年次のPythonは定着が十分でない前提で、復習から入りGAS/JSへ橋渡しする。目標は「書ける」ではなく「**なんとなく読める**」。
- PRIMMは情報Ⅰと同じ様式に準拠しつつ、本時は **Modifyまで（Makeなし）**。Makeは行わず、読む力の養成に専念する。
- GASは前時（QRコードとWebアプリ）で予告した「動的サイトの中身を書く具体ツール」として位置づける。
- 動画は一般知識に徹し、題材コードのアプリ関連づけは後半のPRIMM活動で行う。
