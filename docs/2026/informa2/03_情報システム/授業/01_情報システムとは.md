# 第1時：情報システムとは

## 本時の目標

- 情報システムを「入力・処理・蓄積・出力」という構成要素の組み合わせとして説明できる。
- 身近な情報システム（POSシステム・ATMなど）が社会でどのような役割を果たしているかを理解する。
- 年間プロジェクト「デジタルスタンプラリー」を一つの情報システムとして捉え、これから自分たちが何を作るのか全体像をつかむ。

## キーワード

| 用語 | 説明 |
|---|---|
| 情報システム | コンピュータ・ネットワーク・データ・人が組み合わさり、目的のために情報を処理・活用する仕組み |
| 入力・処理・蓄積・出力 | 情報システムの基本的な働き。データを受け取り（入力）、加工し（処理）、ためて（蓄積）、結果を出す（出力）流れ |
| ハードウェア | コンピュータ本体・スマホ・サーバなど、情報システムを動かす物理的な機器 |
| ソフトウェア | ハードウェアを動かし、処理の手順を担うプログラムのまとまり |
| データ | システムが処理・蓄積する情報のもと。数値・文字・画像など |
| POSシステム | 店舗のレジで販売情報を記録・集計する情報システム。身近な例 |

## 準備物・教材

- 教師：授業動画（情報システムの構成要素と社会での役割）、スライド、ワークシート（身近なシステムの分解／スタンプラリーの分解）
- 教師：年間プロジェクト「デジタルスタンプラリー」の全体像を示す図（アプリ設計 [03_アプリ設計.md](../03_アプリ設計.md) の §1〜2 を基に作成）
- 生徒：1人1台端末、Google Classroom（ワークシート配布・提出）
- 教科書：情報Ⅱ（東京書籍）第4章

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 単元の入り口として「私たちの生活はどんな仕組みで支えられているか」を問いかける。コンビニの会計・電車の改札など身近な場面を挙げさせる | 正解探しではなく自由に出させる。本時のゴール（情報システムを構造で捉える）を共有 |
| 15分 | 展開①（動画視聴・インプット） | 授業動画を視聴し、情報システムの定義・構成要素（入力・処理・蓄積・出力）、身近な例（POSシステム・ATM・予約システム・交通系IC）、社会での役割、ハードウェア／ソフトウェア／データ／人の関係を学ぶ | 動画はアプリには触れず、一般的な知識として扱う。視聴メモ欄に「入力・処理・蓄積・出力」の例を書き取らせる |
| 15分 | 展開②（確認・深化／アプリと関連づけ） | ①身近なシステム（例：POSシステム）を入力・処理・蓄積・出力に分解する確認 → ②同じ枠組みで「デジタルスタンプラリー」を分解する（QR読取＝入力／スタンプ判定＝処理／スプレッドシート＝蓄積／画面表示＝出力） | 動画で得た枠組みを実物に当てはめる。ここで初めてアプリを提示。難しければ図解と例示で段階的に。少人数の利点を生かしペアや個別で確認 |
| 10分 | 展開②続き（プロジェクト全体像の共有） | 年間プロジェクト「デジタルスタンプラリー」を一つの情報システムとして紹介し、これから各単元・各回で何を作るのかの見通しを共有する | 「自分たちが作る側に回る」ことを意識づける。詳細設計には踏み込まず全体像のみ |
| 5分 | まとめ | キーワードを整理し、情報システムは「目的のために情報を処理する仕組み」であることを確認。次時（ネットワークとクライアント・サーバ）へ接続する | 「スタンプラリーの“入力→処理”はどこで起きているか」を次時の問いとして残す |

## 板書・スライド構成

1. 本時のテーマ：情報システムとは
2. 情報システムの定義（コンピュータ・ネットワーク・データ・人の組み合わせ）
3. 構成要素の流れ図：入力 → 処理 → 蓄積 → 出力
4. 身近な例（POSシステム・ATM・予約システム・交通系IC）を流れ図に当てはめる
5. ハードウェア／ソフトウェア／データ／人の関係
6. ＜深化＞デジタルスタンプラリーの分解：QR読取＝入力／スタンプ判定＝処理／スプレッドシート＝蓄積／画面＝出力
7. 年間プロジェクト全体像と本単元の位置づけ

## 本時の評価

- **観点：知識・技能**（評価方法：ワークシート・授業内の発言）
  - 情報システムを「入力・処理・蓄積・出力」の構成要素で説明できているか。
  - 身近なシステムやスタンプラリーを、その枠組みに当てはめて分解できているか。
- 形成的評価とし、単元末の単元テストにつなげる。

## 備考・メモ

- 前単元（情報社会）の評価で「概念理解は得意・手順の順序づけに弱さ」が見られたため、構成要素の流れは図解で順を追って提示する。
- 動画は一般知識に徹し、アプリとの関連づけは後半の確認・深化活動で行う構成とする。
