# 第16時：単元テストと振り返り

## 本時の目標

- 単元で学んだ知識（情報システムの構成・ネットワーク・データ管理・QR/Webアプリ・プログラミングの基礎・API/生成AIの活用・システム設計）を単元テストで確認する。
- 単元全体を振り返り、自分の学びと次への課題を整理する。

## キーワード

| 用語 | 説明 |
|---|---|
| 情報システム | コンピュータ・ネットワーク・データ・人が組み合わさり、目的のために情報を処理・活用する仕組み |
| クライアントとサーバ | サービスを要求する側と、応えて提供する側の役割分担 |
| リレーショナルデータベース（RDB） | テーブル同士をリレーションでつないで管理し、整合性を保つデータベース |
| 検証 | 生成AIの出力をそのまま信じず、読んで・動かして・直して確かめること |

## 準備物・教材

- 教師：単元テスト（選択・記述混合形式。座学01〜07の内容を中心に出題）、振り返りシート
- 教師：採点基準
- 生徒：1人1台端末または紙の解答用紙、筆記用具
- 教科書：情報Ⅱ（東京書籍）第4章

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 出題範囲と形式（選択・記述）を確認する | 範囲は座学で扱った概念が中心であることを伝え、落ち着いて取り組ませる |
| 30分 | 単元テスト | 単元テストに取り組む。情報システムの構成・クライアント/サーバ・RDB・QR/Webアプリ・プログラミングの基礎・API/生成AIの検証・システム設計などを問う | 概念理解・記述を中心に評価する。手順を問う設問は図と結びつけて出題し、つまずきに配慮する |
| 10分 | 振り返り | 単元全体（座学＋演習＋運用）を振り返り、できるようになったこと・難しかったこと・次に活かしたいことを整理する | プロジェクトを通した学びを自分の言葉でまとめさせる |
| 5分 | まとめ | 本単元の総括と、次単元（データサイエンス）への接続を確認する | 集めたデータを次単元で分析することを予告し、学びの連続性を示す |

## 板書・スライド構成

1. 本時のテーマ：単元テストと振り返り
2. 出題範囲・形式の確認
3. （テスト中は掲示最小限）
4. 振り返りの観点（できたこと・難しかったこと・次へ）
5. 次単元データサイエンスへの接続

## 本時の評価

- **観点：知識・技能**（評価方法：単元テスト）
  - 情報システム・ネットワーク・データ管理・QR/Webアプリ・プログラミング基礎・API/生成AIの活用・システム設計の知識を説明できているか。
- **観点：主体的に学習に取り組む態度**（評価方法：振り返りシート）
  - 単元の学びを振り返り、次への課題を見いだしているか。
- 単元テストは観点別評価（知識・技能）の総括的評価とする。

## 備考・メモ

- 単元テストは概念理解・記述を主軸とし、手順を問う設問は図解とセットで出題して「手順の順序づけ」のつまずきに配慮する。
- 演習で蓄積した記録シート・振り返りレポートとあわせて、観点別に総合評価する。
