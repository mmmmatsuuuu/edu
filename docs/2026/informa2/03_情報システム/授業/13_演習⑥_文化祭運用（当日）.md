# 第13時：演習⑥｜文化祭運用（当日）

## 本時の目標

- 役割分担に沿って、スタンプラリーを文化祭当日に実際に運用できる。
- 利用状況をモニタリングし、発生したトラブルにチームで対応できる。
- 実物景品の引き換えを、引換QRとスタッフ確認で正しく行える。

## キーワード

| 用語 | 説明 |
|---|---|
| 運用 | 作ったシステムを実際の場面で動かし続けること |
| モニタリング | 稼働状況や利用状況を観察し、異常に早く気づくこと |
| 運用記録 | 当日の出来事・トラブル・対応をその場で残す記録。後の振り返り・分析に使う |

## 準備物・教材

- 教師：稼働中のシステム、各ブースのQR（掲示済み）、スタッフ用引き換えページ、Chromebook、実物景品（生徒会連携分があれば）
- 教師：運用記録シート（時刻・出来事・対応）、トラブル対応の連絡体制
- 生徒：役割ごとの持ち場、端末、運用記録シート
- 教科書：情報Ⅱ（東京書籍）第4章

## 授業展開

> 文化祭当日の実運用のため、時間配分は当日の枠に合わせる。以下は運用の流れの目安。

| 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|
| 運用準備 | 各QRの掲示・読み取り確認、スタッフ用ページのログイン、景品・引き換え場所の準備、役割の最終確認 | 開始前にスタンプ1件・引き換え1件の通し確認をしておく |
| 運用・モニタリング | スタンプ付与・ガチャ・景品引き換えを運用。利用状況（人数・時間帯・在庫）をモニタリングする | 異常（記録されない・在庫の不整合など）に早く気づき、運用記録に残す |
| トラブル対応 | 不具合や来場者の困りごとに、役割分担で対応する。引換QRが読めない場合は手入力フォールバックに切り替える | 完璧でなくてよい。起きた事象と対応を必ず記録に残すことを優先する |
| 景品引き換え | 来場者の引換QRをスタッフがChromebookで読み取り、未引換を確認して景品を渡し、引換済に更新する | 一回限りの引き換え（引換済の再提示は拒否）を徹底。実物景品は在庫を確認しながら |
| 区切り | 運用記録を回収し、収集データの保存を確認する | 翌時のデータ整理に向け、ログとメモをそろえておく |

## 板書・スライド構成

※当日は掲示・口頭での指示が中心。事前共有スライドとして以下を用意：

1. 当日の役割分担（運用係・引き換え係・トラブル対応係）
2. トラブル時の動き（切り分け→対応→記録→必要なら手入力フォールバック）
3. 引き換え手順（引換QR → スタッフ確認 → 引換済）
4. 運用記録シートの書き方

## 本時の評価

- **観点：主体的に学習に取り組む態度**（評価方法：運用記録・活動状況）
  - 役割を果たし、トラブルにも粘り強く対応しようとしているか。
  - 起きた事象と対応を運用記録に残しているか。
- 形成的評価とし、次時のデータ整理・振り返りにつなげる。

## 備考・メモ

- 当日は予期せぬトラブルが学びの中心。「直せたか」より「気づいて記録し対応したか」を見る。
- 収集データ（スタンプ記録・ガチャ履歴・引き換え記録）は、次単元データサイエンスの分析素材になるため確実に保存する。
