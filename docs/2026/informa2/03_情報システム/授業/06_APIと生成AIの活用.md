# 第6時：APIと生成AIの活用

## 本時の目標

- APIが「プログラム同士の窓口」であり、リクエストとレスポンスでやり取りすることを知識として説明できる。
- 生成AIにコードを書かせる作法（明確な指示・出力の検証）を理解する。
- 生成AIに指示して得たコードをサンドボックスで実行し、テスト入力で動作を確認・修正できる。

## キーワード

| 用語 | 説明 |
|---|---|
| API | プログラム同士が機能やデータをやり取りするための窓口・取り決め |
| Web API | ネットワーク越しに（HTTPで）利用するAPI。リクエストを送ると結果が返る |
| リクエスト／レスポンス | API利用時の「要求」と「応答」（第2時のHTTPと接続） |
| 検証（AI出力の確認） | AIが出力したコードをそのまま信じず、読んで・動かして・直して正しさを確かめること |

## 準備物・教材

- 教師：授業動画（APIの概念・リクエスト/レスポンス・Webサービス連携の例／生成AIにコードを書かせる作法。生成AI・プロンプト・ハルシネーションは情報社会の既習内容として軽く復習）、スライド
- 教師：体験演習の手順書（生成AIへの指示例・サンドボックスの使い方・テスト入力例）、ワークシート（プロンプト・実行結果・検証メモの記入欄）
- 生徒：1人1台端末、生成AIツール、**ネット上のコマンドライン型サンドボックス**（標準入出力で実行できる環境）、Google Classroom
- 教科書：情報Ⅱ（東京書籍）第4章

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 前時の復習（コードを読む）。「コードは自分で書く以外にどう用意できるか」を問いかけ、生成AIの活用につなげる | 前時の"読む"と本時の"書かせて検証する"の関係を示す |
| 15分 | 展開①（動画視聴・インプット） | 授業動画を視聴：APIとは（プログラム同士の窓口）・リクエスト/レスポンス・Webサービス連携の例。続いて生成AIにコードを書かせる作法（具体的・明確なプロンプト／出力をそのまま信じず検証する姿勢） | 動画はアプリには触れず一般知識として扱う。APIは知識レベルにとどめる。生成AI・プロンプト・ハルシネーションは情報社会の既習として簡潔に復習 |
| 22分 | 展開②（体験演習・CLIサンドボックス） | 基本アルゴリズム（例：線形探索・整列）を題材に、①生成AIに指示（プロンプト）してコードを得る → ②**ネット上のサンドボックスで実行** → ③テスト入力で動作を確認 → ④意図と違えばプロンプトを直すか一部を修正して再実行。プロンプト・実行結果・検証メモをワークシートに記録 | 標準入出力で完結する題材を選び、アプリ実装には踏み込まない。「出力を鵜呑みにせず必ず実行して確かめる」を徹底。うまくいかない例も検証の学びとして扱い、個別フォローする |
| 8分 | まとめ | キーワード整理。「AIに任せても、最後に動かして確かめるのは自分」という検証の姿勢を確認。次時（システムの全体設計を読む）・以降の演習へ接続 | 演習で自分の担当パーツをAIで作る際の基本姿勢として位置づける |

## 板書・スライド構成

1. 本時のテーマ：APIと生成AIの活用
2. API＝プログラム同士の窓口／リクエスト→レスポンス（第2時と接続）
3. Web APIとWebサービス連携の例
4. 生成AIにコードを書かせる作法：明確な指示／出力を検証する
5. ＜体験＞指示 → サンドボックス実行 → テストで確認 → 修正の流れ
6. 題材：探索・整列などの基本アルゴリズム（標準入出力で完結）

## 本時の評価

- **観点：知識・技能**（評価方法：ワークシート・授業内の活動）
  - APIの概念（プログラム同士の窓口・リクエスト/レスポンス）を説明できているか。
  - 生成AIに指示して得たコードを実行し、テスト入力で動作を確認・修正できているか。
- **観点：主体的に学習に取り組む態度**（評価方法：活動状況）
  - 出力を検証し、うまくいかない場合に粘り強く指示や修正を試そうとしているか。
- 形成的評価とし、以降の演習につなげる。

## 備考・メモ

- サンドボックスはネット上のコマンドライン型サービスを想定するため、題材はGASやスプレッドシートに依存しない**自己完結した基本アルゴリズム**（探索・整列）とする。アプリの実装は演習で扱う。
- APIは本時では知識レベルにとどめ、手を動かす活動は「指示→実行→検証」のループに充てる。
- 生成AI・プロンプト・ハルシネーションは情報社会で既習のためキーワードには含めず、動画で簡潔に復習する。
- 探索・整列は手を動かすための題材という位置づけで、必要に応じてキーワードに昇格できる。
