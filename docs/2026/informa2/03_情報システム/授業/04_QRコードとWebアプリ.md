# 第4時：QRコードとWebアプリ

## 本時の目標

- URLとHTML・CSS・JavaScriptの役割を理解し、Webページがどう作られているかを説明できる。
- 静的サイトと動的サイト（Webアプリ）の違いを説明できる。
- QRコードがURLを埋め込んだものであり、URLパラメータで違う情報を渡せることを理解し、スタンプラリーのブースごとのQRの仕組みを読み解ける。

## キーワード

| 用語 | 説明 |
|---|---|
| URL | Web上の場所（リソース）を示す住所のような文字列 |
| HTML | Webページの構造・内容を記述する言語 |
| CSS | Webページの見た目（色・配置・装飾）を指定する言語 |
| JavaScript（JS） | Webページに動きや処理を加えるプログラミング言語 |
| 静的サイト | あらかじめ用意された内容をそのまま表示するWebサイト（HTML・CSS中心） |
| 動的サイト（Webアプリ） | アクセスや入力に応じて内容が変わるサイト。ブラウザ上で動くアプリ |
| URLパラメータ | URLの末尾に付けて情報を渡す部分（例：`?booth=A1`）。動的サイトが受け取って表示内容を変えられる |
| QRコード | 二次元に情報を持つコード。URLなどを埋め込め、カメラで読み取れる |
| 誤り訂正 | QRコードの一部が汚れ・破損しても読み取れるようにする仕組み |

## 板書・スライド構成

1. 本時のテーマ：QRコードとWebアプリ
2. URL＝Web上の住所
3. HTML（構造）・CSS（見た目）・JS（動き）の役割分担
4. 静的サイト と 動的サイト（Webアプリ）の違い
5. URLパラメータ：`https://example.com/page?key=value`（動的サイトへの入力）

7. ＜深化＞ブースごとのQR：`…/exec?booth=A1`、boothで判別／スタンプラリー＝動的サイト

## 備考・メモ

- GAS（Google Apps Script）は具体ツールとして次時「プログラミングの基礎」で導入する方が理解しやすいため、本時では「動的サイト＝Webアプリ」までに留める。
- JSは本時で概要（動きを加えるもの）、次時で構文を扱う。軽い重複は定着に有効。
- 動画は一般知識に徹し、アプリとの関連づけは後半の確認・深化活動で行う。
