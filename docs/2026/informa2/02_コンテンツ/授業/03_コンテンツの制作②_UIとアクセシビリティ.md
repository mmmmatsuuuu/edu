# 第3時：コンテンツの制作②：UIとアクセシビリティ

## 本時の目標

- ユーザーインターフェース（UI）とUXの概念を理解し、良いUIの条件を説明できる
- アクセシビリティとユニバーサルデザインの違いを理解し、具体的な配慮の方法を挙げられる
- Webアクセシビリティの考え方を理解し、コンテンツ制作に活かす視点を持てる

## キーワード

| 用語 | 説明 |
|---|---|
| ユーザーインターフェース（UI） | ユーザーとシステムの接点。ボタン・メニュー・画面レイアウトなど、操作や情報表示に関わる要素の総称 |
| UX（ユーザーエクスペリエンス） | ユーザーがサービスや製品を使う中で感じる体験全体。使いやすさ・快適さ・満足感などを含む |
| アクセシビリティ | 年齢・障がい・使用環境などの違いに関わらず、誰でも情報やサービスにアクセスできる度合い |
| ユニバーサルデザイン | 年齢・障がいの有無・文化的背景に関わらず、あらゆる人が使えるよう最初から設計するという考え方 |
| Webアクセシビリティ | Webサイトやアプリを、視覚・聴覚・運動機能に障がいのある人でも利用できるようにする取り組み（代替テキスト・キーボード操作対応など） |

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 「使いにくいと感じたアプリやWebサイトを思い出してみよう」をペアで共有する。どこが使いにくかったかを一言で言語化させる | 「ボタンがわかりにくい」「文字が小さい」「どこをタップすればいいかわからない」など具体的な言葉を引き出す。UIとUXへの問いとして自然につなげる |
| 20分 | 展開①（動画） | 教師作成動画を視聴。内容：①UIとUXの違いと関係（UIはUXの一部）②良いUIの条件（一貫性・フィードバック・シンプルさ・エラー防止）③アクセシビリティとユニバーサルデザインの違い（後付け対応 vs 最初から全員向けに設計）④Webアクセシビリティの具体例（代替テキスト・色のコントラスト・キーボード操作・字幕）⑤ガチャ景品を配信するWebページとアクセシビリティ（来場者の多様性を意識したUI設計） | 動画約12分。UIとUXは「UIは画面の部品、UXはそれを使ったときの感じ方全体」と対比して説明する。アクセシビリティとユニバーサルデザインは「対象を絞るか・最初から全員を想定するか」の違いを強調する |
| 20分 | 展開②（評価・設計活動） | ワークシートに取り組む。①提示された2つのUI画面（良い例・改善が必要な例）を比較し、どこに問題があるかを書き出す②「色だけで情報を伝えている」「文字が小さすぎる」などのアクセシビリティ上の問題を見つけて改善案を書く③ガチャ景品を配信するWebページを「来場者全員が使いやすい」視点で設計するとしたら、どんなUI/UX上の配慮が必要かを記述する（例：ボタンの大きさ・文字のコントラスト・操作ステップ数・エラー時のメッセージなど） | ①はUIの「一貫性」「フィードバック」「シンプルさ」の観点でチェックさせる。②は具体的な改善案（「文字を大きくする」「色に加えてアイコンでも示す」）まで書かせる。③は「景品コンテンツ自体」ではなく「配信Webページの構成と操作性」に焦点を当てることを明示する |
| 5分 | まとめ | 振り返りフォームに記入：①UIとアクセシビリティで最も大事だと思った観点②次の演習でガチャ景品コンテンツを作るときに意識したいこと | 次時（コンテンツの発信）に進む前に、演習では「誰に・何を・どう伝えるか」を設計から始めることを予告する |

## 板書・スライド構成

（動画内のスライド構成）
1. 本時のめあて・キーワード一覧
2. UIとUXの関係図（UIはUXを構成する要素のひとつ）
3. 良いUIの4条件（一貫性・フィードバック・シンプルさ・エラー防止）と具体例
4. アクセシビリティとユニバーサルデザインの違い（比較図）
5. Webアクセシビリティの具体例（代替テキスト・コントラスト・キーボード操作・字幕）
6. ガチャ景品配信WebページのUI設計と多様な来場者（子ども・高齢者・外国人・視覚障がい者など）
7. 本時のまとめ：「誰でも使える・伝わる」設計がコンテンツの価値を高める

## 本時の評価

- **知識・技能**：ワークシートのUI評価から、良いUIの条件とアクセシビリティ上の問題を正確に識別できているか確認
- **思考・判断・表現**：ガチャ景品コンテンツの設計記述から、多様なユーザーを意識した根拠ある設計ができているか評価
