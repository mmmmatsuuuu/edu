# 第2時：デジタル表現①：アナログ・デジタルと情報量・文字

## 本時の目標

- アナログとデジタルの違いを説明できる
- 情報量の単位（bit・byte）を理解し、2進数・10進数・16進数の相互変換ができる
- 文字コードの仕組みを理解し、文字がデジタルデータとして扱われる仕組みを説明できる

## キーワード

| 用語 | 説明 |
|---|---|
| アナログ | 連続した値で情報を表す方式。温度計の目盛りや音の波形など、自然界の情報は多くがアナログ |
| デジタル | 0と1の離散的な値で情報を表す方式。コンピュータはすべての情報をデジタルで扱う |
| 情報量 | デジタルデータの量を表す尺度。ビットやバイトを単位とする |
| ビット（bit） | デジタルデータの最小単位。0か1かの1桁分の情報量を表す |
| バイト（byte） | 8ビットをまとめた単位。1バイトで2⁸＝256種類の情報を表せる |
| 2進数 | 0と1だけを使って数を表す記数法。コンピュータ内部の情報表現の基本 |
| 10進数 | 0〜9の10種類の数字を使う、日常生活で使われる記数法 |
| 16進数 | 0〜9とA〜Fの16種類の記号を使う記数法。データの表記に広く使われる |
| 文字コード | 文字に番号を割り当ててデジタルデータで管理する仕組み。ASCIIやUnicodeが代表例 |

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 「アナログ時計とデジタル時計の違いは何か」を問いかける | 「連続か離散か」という本質的な違いに気づかせる出発点として使う |
| 20分 | 展開①（動画） | 教師作成動画を視聴。内容：①アナログとデジタルの違い（連続値 vs 離散値）②情報量の単位（bit・byte・KB・MB・GB）③2進数・10進数・16進数の仕組みと相互変換④文字コードの概念（ASCIIの成り立ち・Unicodeの役割） | 動画約12分。進数変換は「位取り」の考え方から丁寧に解説する。文字化けの原因（文字コードの不一致）を例に挙げると実感しやすい |
| 20分 | 展開②（計算練習） | ワークシートに取り組む。①2進数⇔10進数の変換（4〜8ビット程度）②10進数⇔16進数の変換③「A」「Z」「0」などの文字がASCIIコードで何番か確認④「1バイトで何種類の文字を表せるか」を計算 | 計算手順を黒板やスライドで示しながら進める。16進数は「A=10、F=15」の対応を先に整理させる |
| 5分 | まとめ | 振り返りフォームに記入：①理解できた点②計算で難しかった箇所 | 次時（音・画像のデジタル表現）への橋渡しとして「音や画像もデジタルに変換できる」ことを予告する |
