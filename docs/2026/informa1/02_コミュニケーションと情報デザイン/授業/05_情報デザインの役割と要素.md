# 第5時：情報デザインの役割と要素

## 本時の目標

- 情報デザインの役割（抽象化・可視化・構造化）を理解し、具体例で説明できる
- 色・形・レイアウトの構成要素と、それぞれの設計原則を理解できる
- シグニファイヤとユーザビリティの概念を理解し、身の回りの事例と結びつけられる

## キーワード

| 用語 | 説明 |
|---|---|
| 抽象化 | 伝えたい情報から本質的な要素だけを取り出し、不要な詳細を省くこと |
| 可視化 | 数値・関係・状態などを図・グラフ・アイコンなどの視覚的な形式に変換すること |
| 構造化 | 情報を階層・順序・グループなどに整理して、全体の関係を明確にすること |
| 色（色相・明度・彩度・配色） | 色を構成する3属性（色相＝色の種類、明度＝明暗、彩度＝鮮やかさ）と、複数の色の組み合わせ方（配色） |
| 形（アイコン・ピクトグラム） | 意味を視覚的に伝えるための図形記号。アイコンはUI上の操作要素、ピクトグラムは誰でも直感的に理解できる絵文字的な記号 |
| レイアウト（近接・整列・反復・対比） | 情報の配置を決める4つの原則。近接（関連要素をまとめる）・整列（要素を揃える）・反復（統一感を持たせる）・対比（重要な要素を際立たせる） |
| シグニファイヤ | 「どう使うか」を視覚的に示す手がかり。ボタンの形状、ドアのグリップなど、操作方法を伝えるデザイン上の手がかり |
| ユーザビリティ | 使いやすさ。目的のタスクを効率よく・正確に・快適に達成できるかを表す設計の品質指標 |

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 「同じ内容でも、伝わるデザインと伝わらないデザインがあるのはなぜか」を問いかけ、2種類の案内板・ポスターを比較して見せる | 「情報が整理されているか」「視覚的にわかりやすいか」の観点を先に提示し、動画の学習内容と接続させる |
| 20分 | 展開①（動画） | 教師作成動画を視聴。内容：①情報デザインの役割（抽象化・可視化・構造化）②色の3属性（色相・明度・彩度）と配色の考え方（補色・同系色）③形の役割（アイコン・ピクトグラムの具体例）④レイアウト4原則（近接・整列・反復・対比）⑤シグニファイヤとアフォーダンスの違い⑥ユーザビリティとは（有効性・効率性・満足度） | 動画約12分。「なぜこのデザインが伝わるのか」を各原則で言語化できるよう丁寧に解説する。ピクトグラムは東京オリンピックのものを例示すると身近に感じやすい |
| 20分 | 展開②（観察・分類活動） | 観察シートに取り組む。①スライドで示す5つのデザイン事例について、使われている情報デザインの要素（色・形・レイアウト）を書き出す②「シグニファイヤがうまく機能している事例」と「機能していない事例」を身の回りから各1つ考えて書く③レイアウト4原則のうち、特に重要だと思うものを1つ選び、理由とともに説明する | ②は「ドアの引き/押し間違い」などの失敗事例を例示してから考えさせると書きやすい。③は正解を求めず、根拠ある選択を重視する |
| 5分 | まとめ | 振り返りフォームに記入：①最も印象に残ったデザイン原則②次時の演習に向けて「観察してみたいもの」を書く | 次時（演習①：身の回りの情報デザインの観察と分析）の準備として、日常のデザインに意識を向けるよう促す |

## 板書・スライド構成

（動画内のスライド構成）
1. 本時のめあて・キーワード一覧
2. 情報デザインの役割：抽象化・可視化・構造化（各1スライドで具体例つき）
3. 色の3属性（色相環・明度・彩度の図解）
4. 配色の考え方（同系色・補色・アクセントカラー）
5. 形の役割：アイコンとピクトグラムの具体例
6. レイアウト4原則（各原則をビフォー/アフターで比較）
7. シグニファイヤとは（良い例・悪い例の比較）
8. ユーザビリティの3要素（有効性・効率性・満足度）
9. 本時のまとめ：「伝わるデザイン」を支える要素

## 本時の評価

- **知識・技能**：観察シートの要素分類から、色・形・レイアウトの概念を正確に理解しているか確認
- **思考・判断・表現**：シグニファイヤの事例記述やレイアウト原則の選択理由から、情報デザインの観点で事例を分析・評価できているか評価
