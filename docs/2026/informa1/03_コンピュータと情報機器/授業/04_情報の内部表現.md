# 第4時：情報の内部表現

## 本時の目標

整数・負の数・小数がコンピューター内部でどのように2進数で表現されているかを理解する。浮動小数点数の仕組みを理解し、誤差（丸め誤差・桁落ち・情報落ち・オーバーフロー）の種類と原因を説明できる。

## キーワード

| 用語 | 説明 |
|---|---|
| 2進数 | 0と1の2種類の数字だけを使う記数法。コンピューター内部のデータ表現の基本 |
| 2の補数 | 負の整数を2進数で表現する方法。元の数のビットを反転して1を加えたもの |
| 浮動小数点数 | 小数を「符号・仮数・指数」に分けて表現する形式。非常に大きな数や小さな数も扱える |
| 丸め誤差 | 有限ビットで小数を表現するときに生じる、端数の切り捨て・切り上げによる誤差 |
| 桁落ち | 値が近い2数を引き算したとき、有効桁数が大幅に減少する現象 |
| 情報落ち | 絶対値の大きな数と小さな数を加算したとき、小さな数が無視されてしまう現象 |
| オーバーフロー | 計算結果が表現できる数の上限を超えてしまうこと |
| アンダーフロー | 計算結果が表現できる数の下限（絶対値が最小の正数）を下回り、0として扱われてしまうこと |

## 板書・スライド構成

1. 今日の目標
2. 復習：整数の2進数表現（前単元で学んだ10進数⇔2進数の変換手順の確認）→「では負の数・小数はどう表現するか？」という問いかけ
3. 負の数の表現：2の補数（ビットを反転して1を加える手順・正の数との対応例）
4. 小数の表現①：固定小数点数（小数点の位置を固定して表現する方法・日常的な小数表記との対応）
5. 小数の表現②：浮動小数点数（符号・仮数・指数の3フィールド構造図・非常に大きな数・小さな数も扱える理由）
6. 誤差：メモリの幅の限界による切り捨て（有限ビットで表しきれない値が丸められることで誤差が生じる仕組み）→ 誤差の種類（丸め誤差・情報落ち・桁落ち・オーバーフロー・アンダーフロー）と具体例
7. まとめ：有限ビット表現の限界と対処の考え方
