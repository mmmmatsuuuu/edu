# 第1時：コンピューターの構成

## 本時の目標

コンピューターを構成するハードウェアの種類と役割を説明できる。CPUの動作サイクル（フェッチ・デコード・実行）を理解する。パーソナルコンピューター以外の情報機器（組み込みシステムなど）の具体例を挙げられる。

## キーワード

| 用語 | 説明 |
|---|---|
| ハードウェア | コンピューターを構成する物理的な機器・部品の総称 |
| CPU（中央処理装置） | プログラムの命令を読み取り、演算・制御を行うコンピューターの中枢部品 |
| GPU（グラフィックス処理装置） | 画像描画や並列演算を高速に処理する専用プロセッサ。AIや科学計算にも活用される |
| 主記憶装置（メモリ） | CPUが直接読み書きする一時的なデータの保存場所。電源を切ると消える |
| 補助記憶装置 | データを長期的に保存する装置（HDD・SSD・USBメモリなど） |
| 入力装置・出力装置 | コンピューターにデータを入力する装置（キーボード等）と結果を出力する装置（ディスプレイ等） |
| インターフェース | 機器や装置同士を接続してデータをやり取りするための規格・仕組み（USB・HDMIなど） |
| フェッチ・デコード・実行 | CPUが命令を「読み込む→解釈する→実行する」という動作サイクル |
| 組み込みシステム | 家電・自動車・医療機器などの特定用途に組み込まれたコンピューターシステム |

## 準備物・教材

- 教科書：情報Ⅰ Flex 第3章
- 授業動画（ハードウェア構成・CPUの動作サイクル・組み込みシステム）
- スライド（スマホスペック具体例・組み込みシステムの事例写真）
- ワークシート（各構成要素の役割を穴埋めで確認、スマホスペック記入欄）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 10分 | 導入 | 「スマートフォン・炊飯器・自動車に共通するものは？」と問いかけ、コンピューターが身の回りに広く使われていることを確認する | 生徒の身近な例から引き出し、本時の学習への関心を高める |
| 20分 | 展開①（動画視聴） | 授業動画を視聴し、コンピューターの5大装置の役割・「入力→記憶→演算→出力」のデータの流れ・主記憶と補助記憶の違い・インターフェースの種類・CPUの性能指標（クロック周波数・コア数）・CPUとGPUの違い・CPUの動作サイクル（フェッチ→デコード→実行）を学ぶ | 視聴中はワークシートの空欄を埋めながら見るよう指示する。データの流れのフロー図は板書に残して後の活動で参照できるようにする |
| 15分 | 展開②（活動） | スマホのスペック表の具体例（機種名・CPU・コア数・クロック周波数・RAM・ストレージ・ディスプレイ解像度・バッテリー容量など）をスライドで1例示す。続けて生徒自身のスマートフォンで仕様ページを調べ、同じ項目をワークシートの表に書き込む。記入後、隣の生徒と比べて気づいたことを共有する | 端末を持っていない生徒には教師が用意した機種例のプリントを渡す。「どのスペックが何の装置に対応するか」を5大装置と対応させながら確認させる |
| 5分 | まとめ | 組み込みシステム・IoT機器（炊飯器・信号機・自動車ECUなど）の事例写真を紹介し、コンピューターの多様性を確認する。ワークシートを保管するよう指示し、次時（ソフトウェア）への橋渡しとして「ハードウェアを動かす仕組みは何か？」と問いかける | 導入の問いと結びつけて「最初に挙げた炊飯器・自動車にもコンピューターが入っている」と締めくくる |

## 板書・スライド構成

1. 今日の目標
2. 身の回りのコンピューター（導入の問いかけ）
3. コンピューターの5大装置（演算・制御・記憶・入力・出力）の図
4. データの流れ（入力装置→主記憶装置→演算装置→出力装置 の矢印付きフロー図）
5. 主記憶と補助記憶の比較表（速度・容量・揮発性）
6. CPUの性能の見方（クロック周波数：命令処理の速さ／コア数：並列処理の多さ）
7. CPUとGPUの違い（逐次処理 vs 大規模並列処理、用途の比較）
8. CPUの動作サイクル（フェッチ→デコード→実行）フロー図
9. インターフェースの種類（USB・HDMI・無線など）
10. スマホスペックの具体例（機種名・CPU・コア数・クロック周波数・RAM・ストレージなど一覧表）
11. 組み込みシステムの事例写真（炊飯器・信号機・自動車ECUなど）
12. まとめ：本時のキーワード一覧
