# 第2時：ソフトウェアの仕組み

## 本時の目標

ソフトウェアの種類（OS・アプリケーション・デバイスドライバー）とそれぞれの役割を説明できる。OS・アプリケーション・デバイスドライバーの動作関係（階層構造と呼び出しの流れ）を理解する。

## キーワード

| 用語 | 説明 |
|---|---|
| ソフトウェア | コンピューターに動作を指示するプログラムの総称 |
| 基本ソフトウェア | OSの別名。ハードウェアとアプリケーションの両方を管理する土台となるソフトウェア |
| OS（オペレーティングシステム） | ハードウェアとアプリケーションの間を仲介する基本ソフトウェア（Windows・macOS・Linuxなど） |
| アプリケーションソフトウェア | 特定の目的のためにOSの上で動くソフトウェア（ブラウザ・表計算ソフトなど） |
| デバイスドライバー | OSとハードウェアの間を仲介し、機器を制御するための専用プログラム |
| API（アプリケーションプログラミングインターフェース） | アプリケーションがOSやほかのソフトウェアの機能を呼び出すための窓口 |

## 準備物・教材

- 教科書：情報Ⅰ Flex 第3章
- 授業動画（ソフトウェアの種類・役割・階層構造・動作関係）
- スライド（階層構造図・動作フロー図）
- ワークシート（ソフトウェアの種類を分類する問題・動作フローを追う問題）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 8分 | 導入 | 前時のハードウェア内容を簡単に復習する。「ハードウェアだけではコンピューターは動かない。何が必要か？」と問いかけ、ソフトウェアの必要性を確認する | ワークシートを見返しながら前時とのつながりを意識させる |
| 25分 | 展開①（動画視聴） | 授業動画を視聴し、ソフトウェアの種類（基本ソフトウェア・アプリケーション・デバイスドライバー）の役割・具体例、OS・アプリケーション・デバイスドライバーの動作関係（階層構造・APIの役割・呼び出しフロー）を学ぶ | 視聴中はワークシートの空欄を埋めながら見るよう指示する |
| 12分 | 展開②（活動） | ワークシートに取り組む（①ソフトウェアの種類分類問題・②「印刷ボタンを押したときの呼び出しの流れ」など動作フローを追う問題）。完成後、全体で答え合わせをする | 階層構造図を参照しながら解くよう促す。「なぜアプリがハードウェアを直接操作しないのか」を問いかけながら確認する |
| 5分 | まとめ | 三層構造（アプリ→OS→ハードウェア）を板書で整理して確認する。次時（論理回路）への接続として「コンピューターの最も基本的な演算はどう行われるか？」と問いかける | 本時の学習と前時のハードウェアがどう対応するかを結びつけて締めくくる |

## 板書・スライド構成

1. 今日の目標
2. ソフトウェアとは（ハードウェアとの対比）
3. ソフトウェアの三種類（基本ソフトウェア・アプリケーション・デバイスドライバーの名称と一覧）
4. 基本ソフトウェア（OS）：役割・代表例（Windows / macOS / Linux / iOS / Android）・「何でも管理する土台」というイメージ
5. アプリケーションソフトウェア：役割・代表例（ブラウザ・Word・音楽プレーヤーなど）・OSの上で動く点を強調
6. デバイスドライバー：役割・代表例（プリンタードライバー・グラフィックドライバー）・ハードウェアごとに必要な点
7. 階層構造図（アプリ ／ OS ／ デバイスドライバー ／ ハードウェア）
8. APIとは（層間の呼び出し窓口）
9. 動作フロー例：「印刷ボタンを押したとき」の呼び出しの流れ
10. まとめ：三層構造の整理
