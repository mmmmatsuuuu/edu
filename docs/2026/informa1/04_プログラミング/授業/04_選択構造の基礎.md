# 第4時：選択構造の基礎（PRIM）

## 本時の目標

if文1つで条件分岐するコードを、Predict→Run→Investigate→Modify の手順で読み解き、条件や処理を書き換えられるようになる。ゼロから書くのではなく「動くコードを読んで直す」ことに慣れる。

## キーワード

| 用語 | 説明 |
|---|---|
| PRIM | Predict（予想）→Run（実行）→Investigate（調査）→Modify（改変）の学習サイクル。本時はMakeなし |
| 条件式 | 真（True）か偽（False）を返す式（`点数 >= 60` など） |
| 比較演算子 | 大小・等価を判定する記号（`>= <= == !=`） |
| 分岐 | 条件の真偽によって実行する処理を選ぶこと |

## 準備物・教材

- PRIMワークシート（Predict/Run/Investigate/Modify 欄・実行環境リンク付き）※欠席者はこれで自己進行
- 例題コード：点数を入力すると合格／不合格を判定するプログラム
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 本時の例題（合否判定）の課題文を確認し、PRIMの進め方を確認する | 「まず読む・予想する、いきなり書かない」ことを徹底する |
| 10分 | Predict | 配布コードを読み、いくつかの入力（例：70点／50点／60点）で何が出力されるか予想を書く | 実行はまだしない。境界値（ちょうど60点）に注目させる |
| 8分 | Run | 実際に実行し、予想と照合する。ズレた箇所をワークシートに記録する | `>=` と `>` の違いが境界値で効くことに気づかせる |
| 12分 | Investigate | 条件式・比較演算子・if/elseの役割を調べ、コードの各行が何をしているか説明を書く | 「条件が偽のときどこへ進むか」を図で確認する |
| 12分 | Modify | 合格ラインを70点に変える／不合格時のメッセージを変える、など指示された改変を行い実行で確認する | 1か所ずつ変えて実行する習慣をつける。エラーは原因を読む |
| 3分 | まとめ | if文1つの構造を板書整理し、次回の多分岐（elif）を予告する | 境界値の扱いを次回への課題として残す |

## 板書・スライド構成

1. 今日の課題：合否判定
2. PRIMの流れ（Predict→Run→Investigate→Modify）
3. if / else と条件式・比較演算子
4. 境界値（`>=` と `>`）の違い
5. Modify課題と確認の進め方
