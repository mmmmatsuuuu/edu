# 第6時：反復構造の基礎（PRIM）

## 本時の目標

forループの基本構造とカウンタ変数の動きを理解し、繰り返し処理のコードをPredict→Run→Investigate→Modify で読み解いて書き換えられるようになる。

## キーワード

| 用語 | 説明 |
|---|---|
| for文 | 決まった回数・範囲だけ処理を繰り返す構文 |
| range | 繰り返す回数や範囲を指定する仕組み（`range(1, 11)` など） |
| カウンタ変数 | ループのたびに値が変わり、繰り返しを制御する変数 |
| アキュムレータ | 繰り返しの中で値を足し込んでいく変数（合計を求めるなど） |

## 準備物・教材

- PRIMワークシート（実行環境リンク付き）※欠席者はこれで自己進行
- 例題コード：1〜10の合計を求めるプログラム／九九表を作るプログラム
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 「同じ処理を100回書くのは大変」という動機から反復構造を導入する | 第3回の `a = a + 1` のトレースを想起させる |
| 10分 | Predict | 合計を求めるコードを読み、ループ各回でカウンタと合計がどう変わるか予想（トレース）を書く | 出力だけでなく「途中の変数の値」を予想させる |
| 8分 | Run | 実行＋途中経過を表示して予想と照合する | range の範囲（終端が含まれない）に注目させる |
| 12分 | Investigate | forとrange、カウンタ変数、アキュムレータの役割を調べ、合計が積み上がる仕組みを説明する | 九九表のコードでループの中の計算がどう変わるか追わせる |
| 12分 | Modify | 合計の範囲を変える／九九の段を変える／積でなく和に変える、などを行い実行で確認する | 初期値（合計を0から始める）を変えるとどうなるかも試させる |
| 3分 | まとめ | forループとカウンタ・アキュムレータを板書整理し、次回「ループの中に条件を入れる」ことを予告する | アキュムレータパターンが次回の集計につながると伝える。**進度に余裕がある場合は予備回「第6.5時（while文）」へ。実施しない場合のみ、ここで while を5〜10分に圧縮紹介する（下記留意点）** |

## 板書・スライド構成

1. 今日の課題：1〜10の合計／九九表
2. for と range（終端は含まれない）
3. カウンタ変数の動き（トレース）
4. アキュムレータ（合計の積み上げ）
5. Modify課題

## 本時の評価

| 観点 | 評価方法 | 内容 |
|---|---|---|
| 知識・技能 | ワークシート（トレース・Investigate） | forループとカウンタ変数の動きをトレースで説明できる |
| 思考・判断・表現 | ワークシート（Modify） | 範囲や計算内容を意図どおりに書き換えられる |

## 備考・メモ

- **while の扱い（予備回との連動）**：余裕があれば予備回「[第6.5時（while文）](06b_while文.md)」を実施する。実施しない場合は、本時のまとめで5〜10分を使い「①for↔whileの等価性（合計を while でも書ける）／②使い分け（回数が決まる→for、条件で続ける→while）／③無限ループの注意（更新を忘れると止まらない・実演）」の3点だけ圧縮紹介する。目標は「自分で書ける」ではなく「読めて・落とし穴を知っている」状態。
