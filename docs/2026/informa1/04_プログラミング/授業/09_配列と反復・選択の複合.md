# 第9時：配列と反復・選択の複合（PRIM）

## 本時の目標

配列・反復・選択をすべて組み合わせた複合問題（成績データのランク別人数の分類）を、Predict→Run→Investigate→Modify で読み解いて書き換えられるようになる。次回からのMake（ゼロベース生成）の土台を固める。

## キーワード

| 用語 | 説明 |
|---|---|
| 複合問題 | 配列の走査・条件分岐・カウントを組み合わせた問題 |
| 多分岐の集計 | 各要素をelif連鎖で分類し、ランクごとにカウントする処理 |
| 複数カウンタ | ランクの数だけカウンタ変数（または配列）を用意して数える |
| パターンの合流 | これまで学んだ走査・選択・カウントが1つのコードに集まること |

## 準備物・教材

- PRIMワークシート（実行環境リンク付き）※欠席者はこれで自己進行
- 例題コード：成績データ（配列）から各ランク（優・良・可・不可）の人数を分類して数えるプログラム
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 第5回（多分岐）・第7回（カウント）・第8回（走査）が1つの問題に合流することを示す | 「これまでの組み合わせ」であることを強調する |
| 10分 | Predict | ランク分類コードを読み、与えられたデータで各ランクが何人になるか予想する | 各要素がどのランクのカウンタを増やすか追わせる |
| 8分 | Run | 実行して予想と照合し、ズレを記録する | elif連鎖の順序と境界値が効くことを再確認させる |
| 12分 | Investigate | 走査・多分岐・複数カウンタの関係を調べ、全体の処理の流れを図で説明する | 「どの構造がどこで働いているか」を分解して整理させる |
| 12分 | Modify | ランクの閾値を変える／ランクを5段階に増やす／特定ランクのみ抽出する、などを行い確認する | カウンタの追加が必要な改変で、Makeへの橋渡しにする |
| 3分 | まとめ | 複合問題の構造を板書整理し、次回はいよいよ自分で書く（Make）ことを予告する | 「読めたものは書ける」見通しを持たせる |

## 板書・スライド構成

1. 今日の課題：ランク別人数の分類
2. 走査＋多分岐＋カウントの合流
3. 複数カウンタ（ランクごとに数える）
4. 構造の分解（どこで何が働くか）
5. Modify課題と次回（Make）への予告

## 本時の評価

| 観点 | 評価方法 | 内容 |
|---|---|---|
| 知識・技能 | ワークシート（Investigate） | 走査・多分岐・カウントの複合構造を分解して説明できる |
| 思考・判断・表現 | ワークシート（Modify） | 分類条件やカウンタを意図どおりに書き換えられる |
