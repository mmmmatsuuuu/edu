# 第10時：統合問題①（PRIMM・Make込み）

## 本時の目標

既習構造に類似した問題（配列の中から偶数の合計だけを求める）について、PRIMMのフルサイクル（Predict→Run→Investigate→Modify→Make）を通し、自分でコードを書いて（Make）問題を解決できるようになる。

## キーワード

| 用語 | 説明 |
|---|---|
| PRIMM | Predict→Run→Investigate→Modify→Make の学習サイクル。本時はMakeまで行う |
| Make | 既習の構造をもとに、与えられた問題をゼロベースでコードに書き起こす活動 |
| 条件付き集計 | 条件に合う要素だけを足し込む処理（偶数判定＋アキュムレータ） |
| 剰余演算（%） | 割った余りを求める演算。偶数判定（`x % 2 == 0`）に使う |

## 準備物・教材

- PRIMMワークシート（Predict〜Make欄・実行環境リンク付き）※欠席者はこれで自己進行
- 種コード：条件付き集計の類題（第7・9回と構造的に類似）
- Make課題：配列の中から偶数の合計だけを求める
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 3分 | 導入 | これまでのPRIMにMake（自分で書く）が加わることを説明し、本時の課題を確認する | 既習構造の組み合わせで解けることを強調し、不安を下げる |
| 15〜20分 | Predict→Run→Investigate | 類題の種コードを予想・実行・調査する。既習構造のため手早く進め、偶数判定（剰余演算）と条件付き集計の型を確認する | 既習なので深追いせず、Makeに時間を残すことを優先する |
| 25分 | Make | 「偶数の合計を求める」コードを各自で書く。書けたら実行し、いくつかのデータで正しさを確認する | いきなり完成を求めず、まず走査→次に条件→次に集計、と段階的に書かせる。詰まったら種コードのどの部分が使えるか問い返す |
| 7分 | まとめ・共有 | 書けたコードを数例共有し、別解（リスト内包表記など）にも触れる。つまずいた点を全体で確認する | エラーの直し方（読む→1か所直す）を価値づける。欠席者はワークシートで同手順を辿れることを案内する |

## 板書・スライド構成

1. 今日の課題：偶数の合計を求める
2. PRIMM（Make が加わる）
3. 偶数判定：剰余演算 `x % 2 == 0`
4. 書く順序：走査→条件→集計
5. 別解と振り返り

| 主体的に学習に取り組む態度 | 演習活動状況・ワークシート | 試行錯誤しながら粘り強くコードを書き・直そうとしている |
