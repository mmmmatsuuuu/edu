# 第5時：選択構造の複雑化（PRIM）

## 本時の目標

elif連鎖やネストした条件分岐を読み解き、条件の順序が結果を左右することに気づく。多分岐のコードをPredict→Run→Investigate→Modify で扱い、正しく書き換えられるようになる。

## キーワード

| 用語 | 説明 |
|---|---|
| elif | 「そうでなければ、もし〜なら」を表す多分岐の構文 |
| 多分岐 | 3つ以上の場合に処理を分けること |
| ネスト | if文の中にさらにif文を入れる入れ子構造 |
| 条件の順序 | elif連鎖では上から判定されるため、順序を誤ると意図しない分岐になる |

## 準備物・教材

- PRIMワークシート（実行環境リンク付き）※欠席者はこれで自己進行
- 例題コード：点数を5段階（優・良・可・不可・不合格）に分類するプログラム
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 前回のif/elseを振り返り、「3つ以上に分けるには?」という問いから多分岐へ導く | 前回の合否判定を拡張する流れにする |
| 10分 | Predict | 5段階分類コードを読み、複数の点数で予想を書く。境界値も含める | elif連鎖が上から評価されることに注目させる |
| 8分 | Run | 実行して予想と照合し、ズレを記録する | 想定外の分岐が出たら「なぜそこへ流れたか」を考えさせる |
| 12分 | Investigate | elif連鎖の評価順序とネストの動きを調べ、各分岐の到達条件を整理する | 条件を入れ替えると結果が変わる例を示し、順序の重要性に気づかせる |
| 12分 | Modify | 段階の閾値を変える／判定の順序をわざと入れ替えてバグを観察し直す、などを行う | バグを「直す」体験を通じて順序の効果を実感させる |
| 3分 | まとめ | 多分岐と条件の順序を板書整理し、次回の反復構造を予告する | 「順序づけ」が後のテスト（並べ替え）にもつながることを伝える |

## 板書・スライド構成

1. 今日の課題：5段階分類
2. elif連鎖は上から評価される
3. 条件の順序を変えるとどうなるか
4. ネスト（入れ子）の読み方
5. Modify課題

|---|---|---|

| 思考・判断・表現 | ワークシート（Modify） | 条件の順序を意識して多分岐を正しく書き換えられる |
