# 第7時：反復と選択の複合（PRIM）

## 本時の目標

ループの中にif文を組み込んだコード（カウント・集計）を読み解き、カウンタ／アキュムレータパターンを理解する。Predict→Run→Investigate→Modify で条件付きの繰り返しを書き換えられるようになる。

## キーワード

| 用語 | 説明 |
|---|---|
| 複合構造 | 反復の中に選択を入れるなど、基本構造を組み合わせた構造 |
| カウントパターン | 条件に合う要素の個数を数える定番処理 |
| アキュムレータパターン | 条件に合う値だけを足し込む定番処理 |
| ネスト（反復＋選択） | for の中に if を入れた入れ子構造 |

## 準備物・教材

- PRIMワークシート（実行環境リンク付き）※欠席者はこれで自己進行
- 例題コード：リストの中から60点以上の人数を数えるプログラム
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 前回の合計（アキュムレータ）と第4・5回の条件分岐を組み合わせる、という見通しを示す | 「反復＋選択」が今後の演習の中心になると伝える |
| 10分 | Predict | 60点以上を数えるコードを読み、与えられたリストで何人になるか・各回でカウンタがどう変わるか予想する | 条件に合うときだけカウンタが増えることに注目させる |
| 8分 | Run | 実行＋途中表示で予想と照合する | 境界値（ちょうど60点）が条件に含まれるか確認させる |
| 12分 | Investigate | forの中のifの動き、カウンタ変数の増え方を調べ、処理の流れを図で説明する | 「ループのどの位置でカウントしているか」を意識させる |
| 12分 | Modify | 条件を「80点以上」に変える／人数でなく合計点に変える／平均を出す、などを行い確認する | カウントと集計（合計）の違いを意識させる。1か所ずつ変える |
| 3分 | まとめ | カウント／アキュムレータパターンを板書整理し、次回の配列操作を予告する | このパターンが配列処理で繰り返し現れることを伝える |

## 板書・スライド構成

1. 今日の課題：60点以上の人数を数える
2. for の中に if を入れる（複合構造）
3. カウントパターン／アキュムレータパターン
4. 境界値と条件の確認
5. Modify課題

## 本時の評価

| 観点 | 評価方法 | 内容 |
|---|---|---|
| 知識・技能 | ワークシート（Investigate） | ループ内の条件分岐とカウント／集計の仕組みを説明できる |
| 思考・判断・表現 | ワークシート（Modify） | 条件や集計内容を意図どおりに書き換えられる |
