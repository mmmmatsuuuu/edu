# 第1時：プログラミングとアルゴリズム

## 本時の目標

プログラミングとは何か、プログラミング言語にはどのような種類（高級言語／低級言語、コンパイル型／インタプリタ型）があるかを理解する。身近な問題を解くための手順（アルゴリズム）をフローチャートで表現できるようになり、コードを書く前に「手順を考える」段階があることを実感する。

## キーワード

| 用語 | 説明 |
|---|---|
| プログラム | コンピュータに行わせる処理の手順を、コンピュータが理解できる形で記述したもの |
| プログラミング言語 | プログラムを記述するための人工言語。用途や特徴により多くの種類がある |
| 高級言語 | 人間に理解しやすい形で書ける言語（Python・JavaScript など）。記述が容易 |
| 低級言語 | 機械語に近い言語（機械語・アセンブリ言語）。ハードウェアを直接的に制御できる |
| コンパイル型 | プログラム全体を事前に機械語へ翻訳（コンパイル）してから実行する方式。実行が速い |
| インタプリタ型 | プログラムを1行ずつ翻訳しながら実行する方式。手軽に試せる（Python など） |
| アルゴリズム | 問題を解決するための手順・段取り。同じ問題でも複数のアルゴリズムがありうる |
| フローチャート | 処理の流れを図記号（端子・処理・判断・繰り返し）で表したもの |
| 順次・選択・反復 | あらゆるアルゴリズムを構成する3つの基本構造 |

## 準備物・教材

- 教科書：情報Ⅰ Flex 第4章
- 授業動画（プログラミングとは／プログラミング言語の種類〈高級・低級／コンパイル型・インタプリタ型〉／アルゴリズム／フローチャートの読み書き）※展開①／欠席者対応
- 視聴用ワークシート（言語の分類・図記号・流れの空欄）
- スライド（言語の分類図・フローチャートの図記号・身近な例）
- アンプラグド演習シート（お釣りの計算・じゃんけんの判定などをフローチャートに落とす）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 10分 | 導入 | 「コンピュータは指示された手順しか実行できない」ことを、身近な自動化（自動ドア・券売機）を例に確認する。プログラム＝手順の記述であることを押さえる | まだコードは見せない。「曖昧な指示では動かない」体験を重視する |
| 15分 | 展開①（動画視聴） | 授業動画を視聴し、(1)プログラミング言語の種類——高級言語／低級言語、コンパイル型／インタプリタ型（本単元はインタプリタ型のPythonを使うこと）、(2)アルゴリズムとフローチャートの図記号（端子・処理・判断・反復）と読み書きを学ぶ。簡単な例（数を2倍して表示）のフローチャート化も動画で確認する | 視聴中はワークシートの空欄（言語の分類・図記号・流れ）を埋めながら見るよう指示する。言語の分類は「身近なPythonがどこに位置するか」を結びつけて押さえさせる |
| 18分 | 展開②（アンプラグド演習） | シートの問題（例：お釣りの計算、じゃんけんの勝敗判定）を、コードを使わずフローチャートで表現する。ペアで見せ合い、流れが追えるか確認する | じゃんけんは「判断」が複数必要なことに気づかせる。正解は1つではないことを伝える |
| 7分 | まとめ | 作成したフローチャートを数例共有し、「順次・選択・反復」という3つの形が現れていることを予告する。次回からコードと対応づけることを伝える | 自分の手順を他者が再現できるか、という観点で振り返らせる |

## 板書・スライド構成

1. 今日の目標
2. プログラム＝手順の記述（身近な自動化の例）
3. プログラミング言語の分類（高級／低級・コンパイル型／インタプリタ型、Pythonの位置）※動画と対応
4. フローチャートの図記号（端子・処理・判断・反復）※動画と対応
5. 例：数を2倍して表示するフローチャート
6. アンプラグド演習（お釣り・じゃんけん）
7. まとめ：3つの基本構造の予告
