# 第8時：配列の基礎操作（PRIM）

## 本時の目標

配列を先頭から順に調べる（走査する）処理を理解し、最大値・最小値・合計・平均を求めるコードをPredict→Run→Investigate→Modify で読み解いて書き換えられるようになる。

## キーワード

| 用語 | 説明 |
|---|---|
| 走査（イテレーション） | 配列の要素を先頭から順に1つずつ取り出して処理すること |
| 最大値・最小値 | 配列の中で最も大きい／小さい値。暫定値と比較しながら更新する |
| 合計・平均 | アキュムレータで足し込み、要素数で割って平均を求める |
| 暫定値 | 探索の途中で「今のところの最大／最小」として保持する値 |

## 準備物・教材

- PRIMワークシート（実行環境リンク付き）※欠席者はこれで自己進行
- 例題コード：配列の最大値・最小値を探すプログラム／合計・平均を求めるプログラム
- オンライン実行環境（Google Colab 等）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 第6・7回の反復を「配列の各要素に対して行う」と捉え直し、走査の見通しを示す | 第3回の配列・インデックスを想起させる |
| 10分 | Predict | 最大値を求めるコードを読み、走査の各回で暫定最大がどう更新されるか予想（トレース）する | 「より大きい値が来たら更新」という比較の流れに注目させる |
| 8分 | Run | 実行＋途中表示で予想と照合する | 暫定値の初期値の取り方（先頭要素にする等）を確認させる |
| 12分 | Investigate | for による走査、暫定値の更新、合計／平均の求め方を調べ、流れを説明する | 最大値・合計が「ループの中で少しずつ確定する」点を強調する |
| 12分 | Modify | 最大値を最小値に変える／平均を求める／合計を出す、などを行い確認する | 比較の向き（`>` と `<`）を変える効果を実感させる |
| 3分 | まとめ | 走査と暫定値・集計を板書整理し、次回「配列＋反復＋選択の複合」を予告する | これまでの要素が次回ですべて合流すると伝える |

## 板書・スライド構成

1. 今日の課題：最大・最小／合計・平均
2. 配列の走査（先頭から順に）
3. 暫定値の更新（最大値の探し方）
4. 合計→平均の求め方
5. Modify課題
