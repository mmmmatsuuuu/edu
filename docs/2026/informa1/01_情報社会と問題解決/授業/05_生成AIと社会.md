# 第5時：生成AIと社会

## 本時の目標

- 生成AIの基本的な仕組みと代表的な活用事例を説明できる
- 画像生成AIの体験を通じて、プロンプトの重要性・言語表現の限界・AIの非決定性を実感できる
- 生成AIを使う上で気をつけるべきことを自分の言葉でまとめられる

## 準備物・教材

【教師】
- 授業動画（約12分）：生成AIの仕組み・活用事例・ハルシネーション・著作権問題・倫理的課題
- お題画像（1〜2枚）：複数の要素を含むシンプルな画像（例：「赤い帽子をかぶった白い猫が木の上に座っている」など、細部の描写が必要なもの）
- 気づきまとめシート（Google Classroom 配布）

【生徒】
- 学習用端末
- 画像生成AIへのアクセス（Gemini の画像生成機能）
- 教科書 Flex（生成AIは最新追加コンテンツのため補助プリント等を活用）

## 授業展開

| 時間 | 段階 | 学習活動 | 指導上の留意点 |
|---|---|---|---|
| 5分 | 導入 | 「生成AIを使ったことがある人？どんな使い方をした？」を問いかけ、使用経験を共有 | 使ったことがない生徒が多いことを想定し、「今日が初体験の人が多いと思う。それでいい」と伝える |
| 20分 | 展開①（動画） | 教師作成動画を視聴。内容：①生成AIとは（LLM・画像生成・音楽生成の概要）②主な活用事例（ビジネス・教育・医療）③ハルシネーションとは（事例紹介）④著作権的課題（学習データの問題・生成物の著作権）⑤倫理的課題（バイアス・なりすまし・ディープフェイク） | 動画約12分。「次の活動でこれらを体験する」という見通しを持ちながら視聴させる |
| 20分 | 展開②（画像生成チャレンジ） | 【タスク①：お題画像の再現】教師が提示したお題画像と同じ画像を画像生成AIで再現することに挑戦する。①特徴をテキスト（プロンプト）で表現②AIに入力して結果を確認③お題と異なる点を記録しプロンプトを改善して再挑戦（約10分）。【タスク②：職業バイアスの発見】「社長」「看護師」「エンジニア」の画像を生成し、出力の傾向（性別・年齢・人種等）をメモする。複数の職種を比べてパターンを書き出す（約8分）。④気づきまとめシートに記入：「うまくいかなかった理由」「タスク②で気づいた傾向とその原因」「AIを使う上で気をつけるべきこと」 | タスク①：うまく再現できないことが学習の核心。「失敗した」ではなく「限界を発見した」と伝える。タスク②：「なぜ社長はこうなるのか」を問い、学習データの偏りという概念に自然につなげる。正解を言わず、生徒に気づかせる |
| 5分 | まとめ | 生徒数人に「何が難しかったか・何に気づいたか」を発表させる。振り返りフォームに「生成AIを使う上で気をつけたいこと」を記入 | 「言葉で正確に伝えることの難しさ」「AIの出力は確認が必要」「著作物の再現リスク」などが出てくることを期待。出なければ教師が補足する |

## 体験から引き出したい気づき

| 体験した現象 | 結びつけたい概念 |
|---|---|
| 同じプロンプトでも毎回違う結果が出る | AIの非決定性・出力の不安定さ |
| 細かい特徴（色・位置・表情）が伝わらない | 言語表現の限界・プロンプトエンジニアリングの重要性 |
| 指示していない要素がAIに追加される | ハルシネーション（視覚版）・AIの推測の問題 |
| 有名キャラに似た画像が生成できてしまう | 著作権・ディープフェイクのリスク |
| 「社長」が中年男性、「看護師」が女性になりやすい | 学習データのバイアス・社会的ステレオタイプの再現 |
| 思い通りにならないのに使える場面はある | AIは「道具」であり、使い手の判断が重要 |

## 本時の評価

- **知識・技能**：気づきまとめシートの記述から、生成AIの特性・課題を理解しているか確認
- **思考・判断・表現**：「うまくいかなかった理由」の記述から、体験と概念を結びつけて考えられているか評価
- **主体的に学習に取り組む態度**：プロンプトを試行錯誤する姿勢・「失敗」から学ぼうとしているか確認
